# queryPage 接口文档

> 作者：小猫会发光
> 日期：2026-05-04

通用说明：

- 所有接口均为 `POST` 方法，`Content-Type: application/json`。
- 接口统一返回 `HttpResult<Page<T>>` 包装结构。
- `current` 默认 `1`，`size` 默认 `10`（在 Service 层兜底）。

---

## 通用响应结构

### HttpResult\<T\>

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| code | long | 状态码，`200` 成功，`-1` 失败 |
| data | T | 响应数据，分页接口为 `Page<T>` |
| msg | String | 响应消息 |
| timestamp | long | 服务端时间戳（毫秒） |
| refreshToken | String | 续签 token，可为空 |

### Page\<T\>

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| records | List\<T\> | 当前页记录 |
| total | long | 总记录数 |
| size | long | 页大小 |
| current | long | 当前页码 |

---

## 一、用户管理 - 用户列表

### 基本信息

| 项 | 值 |
| --- | --- |
| 接口路径 | `/user/queryPage` |
| 请求方式 | `POST` |
| Content-Type | `application/json` |
| Tag | 用户管理 |
| Operation | 用户列表 |

### 请求参数（UserPageParam）

继承自 `PageParam`。

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| current | long | 否 | 页码，默认 1 |
| size | long | 否 | 页大小，默认 10 |
| search | String | 否 | 搜索关键字，匹配 `username` / `nickname` / `id` 模糊查询 |
| roleId | String | 否 | 角色 id，按角色过滤 |
| orgId | String | 否 | 机构 id，按机构过滤 |
| type | String | 否 | 账号类型，`0` 用户、`1` 系统 |

### 响应数据（Page\<User\>）

`records` 数组中的元素结构：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | Integer | 用户 id |
| type | String | 用户类型 0 用户 / 1 系统 |
| deleted | String | 删除标识 0 否 / 1 是（逻辑删除字段） |
| username | String | 用户名 |
| password | String | 密码（一般不下发，按需脱敏） |
| nickname | String | 用户昵称 |
| createTime | String | 创建时间 `yyyy-MM-dd HH:mm:ss` |
| updateTime | String | 更新时间 `yyyy-MM-dd HH:mm:ss` |
| clientName | String | 客户端名称 |
| clientId | Integer | 客户端 ID |
| userExtend | UserExtend | 用户扩展信息（可空） |

UserExtend 字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| userId | Integer | 用户 id |
| sex | String | 性别 |
| mail | String | 邮箱 |
| phone | Long | 手机号 |

### 请求示例

```json
{
  "current": 1,
  "size": 10,
  "search": "admin",
  "roleId": "1",
  "orgId": "1",
  "type": "0"
}
```

### 响应示例

```json
{
  "code": 200,
  "msg": "请求成功",
  "timestamp": 1714800000000,
  "refreshToken": null,
  "data": {
    "records": [
      {
        "id": 1,
        "type": "0",
        "deleted": "0",
        "username": "admin",
        "nickname": "管理员",
        "createTime": "2024-07-17 14:12:00",
        "updateTime": "2024-07-17 14:12:00",
        "clientName": "web",
        "clientId": 1,
        "userExtend": {
          "userId": 1,
          "sex": "0",
          "mail": "admin@example.com",
          "phone": 13800000000
        }
      }
    ],
    "total": 1,
    "size": 10,
    "current": 1
  }
}
```

## 三、机构管理 - 机构分页

### 基本信息

| 项 | 值 |
| --- | --- |
| 接口路径 | `/org/queryPage` |
| 请求方式 | `POST` |
| Content-Type | `application/json` |
| Tag | 机构管理 |
| Operation | 分页 |

### 请求参数（OrgPageParam）

继承自 `PageParam`。

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| current | long | 否 | 页码，默认 1 |
| size | long | 否 | 页大小，默认 10 |
| search | String | 否 | 搜索关键字 |
| parentId | String | 否 | 父级机构 id，未传则使用默认顶级父 id（`ORG_PARENT`） |

### 响应数据（Page\<Org\>）

`records` 数组中的元素结构：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | Integer | 组织 id |
| parentId | Integer | 父级机构 id |
| parentName | String | 父级机构名称（非数据库字段，按需回填） |
| name | String | 机构名称 |
| deleted | String | 删除标识 0 否 / 1 是 |
| createTime | String | 创建时间 `yyyy-MM-dd HH:mm:ss` |
| updateTime | String | 更新时间 `yyyy-MM-dd HH:mm:ss` |

### 请求示例

```json
{
  "current": 1,
  "size": 10,
  "search": "技术部",
  "parentId": "0"
}
```

### 响应示例

```json
{
  "code": 200,
  "msg": "请求成功",
  "timestamp": 1714800000000,
  "refreshToken": null,
  "data": {
    "records": [
      {
        "id": 1,
        "parentId": 0,
        "parentName": "顶级机构",
        "name": "技术部",
        "deleted": "0",
        "createTime": "2024-11-29 09:30:00",
        "updateTime": "2024-11-29 09:30:00"
      }
    ],
    "total": 1,
    "size": 10,
    "current": 1
  }
}
```
